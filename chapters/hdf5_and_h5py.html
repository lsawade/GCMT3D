Installation of hdf5 and h5py¶

To enable processing of the ASDF files, specifically windowing of the data, hdf5 and and h5py need to be installed on your machine.

hdf5 on the remote cluster¶

On remote clusters hdf5 is usually precompiled and you don’t have to worry about compilation. On the Princeton Tiger Cluster, the command to load a parallel compiled version of hdf5 would be:

module load hdf5/intel-17.0/openmpi-1.10.2/1.10.0

Usually they are easy to identify since they will have either been configured with openmpi or intel-mpi. Hence, when one checks for available modules using module load. Then, the exact location of the installation can be found using the following line.

module show hdf5/intel-17.0/openmpi-1.10.2/1.10.0

On the Princeton Tiger Cluster the output would look somewhat like this:

prepend-path         LD_LIBRARY_PATH /usr/local/hdf5/intel-17.0/openmpi-1.10.2/1.10.0/lib64

The first line shows the location of the installation: /usr/local/share/Modules/modulefiles/hdf5/intel-17.0/openmpi-1.10.2/1 .10.0. This is important for the next step which is the installation of h5py which in turn is necessary to use parallel processing of the ASDF data sets.

Installation of h5py¶

To install h5py on the cluster, the precompiled version of mpi4py from the server has to be loaded. Make sure it isn’t installed beforehand! Simply

module load mpi4py

After this is load we can install h5py in the parallel version. For this, the python source has to be downloaded because the pip install <package> would install the package. Downloading to the current directory and unpacking is done using:

pip download --no-binary=h5py h5py
tar -xvf h5py-?.*.tar.gz
cd h5py-?.*

Then, make sure that you have your favorite mpicc compiler at hand. Meaning, re-module load an existing mpi compiler because the conda environment overwrites the system compiler.

# E.g.
module load openmpi

Afterwards, the package using following lines install using the following lines:

export CC=mpicc
python setup.py configure --mpi [--hdf5=/path/to/parallel/hdf5]
python setup.py build

where the /path/to/parallel/hdf5 is

/usr/local/share/Modules/modulefiles/hdf5/intel-17.0/openmpi-1.10.2/1.10.0

which is the path we found in the above section.
